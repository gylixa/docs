Админка Django (admin.py)

Основные концепции

admin.py используется для настройки административного интерфейса Django. Он позволяет: - Регистрировать модели для отображения в админке - Кастомизировать внешний вид и поведение моделей - Добавлять поиск, фильтрацию и действия - Контролировать права доступа

1. Базовая регистрация модели

1from django.contrib import admin
2from .models import Product
3
4admin.site.register(Product)

Объяснение: Минимальный код для добавления модели Product в админку. Django создаст стандартный интерфейс для CRUD операций.

2. Кастомизация через ModelAdmin

 1from django.contrib import admin
 2from .models import Product
 3
 4class ProductAdmin(admin.ModelAdmin):
 5    list_display = ['name', 'price', 'in_stock', 'created_at']
 6    list_filter = ['category', 'in_stock']
 7    search_fields = ['name', 'description']
 8    ordering = ['-created_at']
 9
10admin.site.register(Product, ProductAdmin)

Объяснение: ModelAdmin позволяет кастомизировать отображение модели. list_display - поля в списке, list_filter - фильтры справа, search_fields - поиск по указанным полям, ordering - сортировка.

3. Декоратор @admin.register

 1from django.contrib import admin
 2from .models import Category, Product
 3
 4@admin.register(Category)
 5class CategoryAdmin(admin.ModelAdmin):
 6    list_display = ['name', 'product_count']
 7
 8@admin.register(Product)
 9class ProductAdmin(admin.ModelAdmin):
10    list_display = ['name', 'category', 'price']

Объяснение: Декоратор @admin.register - современный способ регистрации моделей. Декоратор принимает модель как аргумент и связывает её с классом ModelAdmin.

4. Раздельная регистрация и кастомизация

 1# Способ 1: Регистрация с кастомизацией сразу
 2admin.site.register(Product, ProductAdmin)
 3
 4# Способ 2: Создание экземпляра ModelAdmin
 5admin.site.register(Product, admin.ModelAdmin)
 6
 7# Способ 3: Декоратор (рекомендуется)
 8@admin.register(Product)
 9class ProductAdmin(admin.ModelAdmin):
10    pass

Объяснение: Django предлагает несколько способов регистрации моделей. Декоратор наиболее читаемый и рекомендуется для нового кода.

5. Удаление регистрации (unregister)

 1from django.contrib import admin
 2from django.contrib.auth.models import User, Group
 3
 4# Удаление стандартных моделей Django
 5admin.site.unregister(User)
 6admin.site.unregister(Group)
 7
 8# Регистрация с кастомизацией
 9@admin.register(User)
10class CustomUserAdmin(admin.ModelAdmin):
11    list_display = ['username', 'email', 'is_staff']

Объяснение: unregister() удаляет модель из админки. Полезно для удаления стандартных моделей (User, Group) перед их кастомизированной перерегистрацией.

6. Групповая регистрация

1from django.contrib import admin
2from .models import Author, Book, Publisher
3
4# Регистрация нескольких моделей с одним ModelAdmin
5class BaseAdmin(admin.ModelAdmin):
6    list_per_page = 20
7    save_on_top = True
8
9admin.site.register([Author, Book, Publisher], BaseAdmin)

Объяснение: Можно зарегистрировать несколько моделей одновременно, передав список. Все модели получат одинаковую конфигурацию ModelAdmin.

7. Встроенные редакторы (inlines)

 1from django.contrib import admin
 2from .models import Author, Book
 3
 4class BookInline(admin.TabularInline):
 5    model = Book
 6    extra = 1
 7
 8@admin.register(Author)
 9class AuthorAdmin(admin.ModelAdmin):
10    inlines = [BookInline]
11    list_display = ['name', 'book_count']

Объяснение: TabularInline и StackedInline позволяют редактировать связанные модели на странице родительской модели. extra определяет количество пустых форм.

8. Административные действия

1@admin.register(Product)
2class ProductAdmin(admin.ModelAdmin):
3    actions = ['make_inactive']
4
5    def make_inactive(self, request, queryset):
6        queryset.update(in_stock=False)
7        self.message_user(request, f"{queryset.count()} products deactivated")
8    make_inactive.short_description = "Mark selected as inactive"

Объяснение: Кастомные действия позволяют выполнять операции с выбранными объектами. queryset содержит выбранные объекты, request - HTTP запрос.

9. Права доступа и безопасность

 1@admin.register(Product)
 2class ProductAdmin(admin.ModelAdmin):
 3    # Кто может видеть модель
 4    def has_module_permission(self, request):
 5        return request.user.is_staff
 6
 7    # Кто может добавлять объекты
 8    def has_add_permission(self, request):
 9        return request.user.is_superuser
10
11    # Кто может изменять объекты
12    def has_change_permission(self, request, obj=None):
13        if obj and obj.created_by == request.user:
14            return True
15        return request.user.is_superuser

Объяснение: Методы has_*_permission контролируют права доступа на уровне модели и объектов. Полезно для сложных систем прав.

10. Кастомизация форм в админке

 1from django import forms
 2from .models import Product
 3
 4class ProductAdminForm(forms.ModelForm):
 5    class Meta:
 6        model = Product
 7        fields = '__all__'
 8        widgets = {
 9            'description': forms.Textarea(attrs={'rows': 4}),
10        }
11
12@admin.register(Product)
13class ProductAdmin(admin.ModelAdmin):
14    form = ProductAdminForm
15    fieldsets = (
16        ('Основное', {'fields': ('name', 'price')}),
17        ('Дополнительно', {'fields': ('category', 'description')}),
18    )

Объяснение: form позволяет использовать кастомную форму в админке. fieldsets организует поля в логические группы с заголовками.

11. Переопределение шаблонов админки

1@admin.register(Product)
2class ProductAdmin(admin.ModelAdmin):
3    change_list_template = 'admin/myapp/change_list.html'
4    change_form_template = 'admin/myapp/change_form.html'
5
6    class Media:
7        css = {'all': ('admin/css/custom.css',)}
8        js = ('admin/js/custom.js',)

Объяснение: change_*_template переопределяют стандартные шаблоны. Класс Media добавляет кастомные CSS/JS файлы.

Регистрация и удаление (register/unregister) - подробнее

Паттерн unregister/register

 1# 1. Удаляем стандартную регистрацию
 2admin.site.unregister(User)
 3
 4# 2. Регистрируем с кастомизацией
 5@admin.register(User)
 6class CustomUserAdmin(UserAdmin):
 7    list_display = ['username', 'email', 'first_name', 'last_name', 'is_active']
 8
 9    # Добавляем кастомное поле в форму
10    fieldsets = (
11        (None, {'fields': ('username', 'password')}),
12        ('Personal info', {'fields': ('first_name', 'last_name', 'email')}),
13        ('Custom field', {'fields': ('custom_field',)}),
14    )

Объяснение: Этот паттерн используется для расширения или изменения поведения стандартных моделей Django (User, Group, Permission).

Динамическая регистрация

 1from django.apps import apps
 2
 3# Автоматическая регистрация всех моделей приложения
 4for model in apps.get_app_config('myapp').get_models():
 5    admin.site.register(model)
 6
 7# Исключение определенных моделей
 8EXCLUDE_MODELS = ['AuditLog', 'SystemConfig']
 9app_models = apps.get_app_config('myapp').get_models()
10
11for model in app_models:
12    if model.__name__ not in EXCLUDE_MODELS:
13        admin.site.register(model)

Объяснение: Автоматическая регистрация всех моделей приложения. Полезно для больших приложений или плагинов.

Множественная админка для одной модели

 1from django.contrib import admin
 2
 3# Первая админка для основной работы
 4@admin.register(Product, site=admin.site)
 5class ProductAdmin(admin.ModelAdmin):
 6    list_display = ['name', 'price']
 7
 8# Вторая админка для архивных записей
 9class ArchivedProductAdmin(admin.ModelAdmin):
10    list_display = ['name', 'archived_at']
11
12    def get_queryset(self, request):
13        return super().get_queryset(request).filter(archived=True)
14
15# Создаем отдельный сайт админки
16archived_admin = admin.AdminSite(name='archivedadmin')
17archived_admin.register(Product, ArchivedProductAdmin)

Объяснение: Одна модель может быть зарегистрирована в нескольких админках с разной конфигурацией. Полезно для разделения функциональности.

Лучшие практики работы с admin.py

  1. Используйте декоратор ``@admin.register`` для читаемости кода

  2. Всегда указывайте ``list_display`` для удобства просмотра списка

  3. Добавляйте поиск и фильтры для больших таблиц

  4. Разделяйте права доступа через has_*_permission методы

  5. Используйте unregister/register для кастомизации стандартных моделей

  6. Тестируйте админку как и остальное приложение

  7. Ограничивайте ``list_per_page`` для производительности

  8. Добавляйте ``prefetch_related`` и ``select_related`` в get_queryset для оптимизации запросов

  9. Используйте ``readonly_fields`` для полей, которые не должны изменяться

  10. Логируйте важные действия через сигналы или переопределенные методы save

Пример полного файла admin.py

 1"""
 2admin.py - Конфигурация административного интерфейса
 3"""
 4
 5from django.contrib import admin
 6from django.contrib.auth.admin import UserAdmin
 7from django.contrib.auth.models import User, Group
 8from .models import Category, Product, Order
 9
10# Удаляем стандартные модели
11admin.site.unregister(User)
12admin.site.unregister(Group)
13
14# Регистрируем кастомного пользователя
15@admin.register(User)
16class CustomUserAdmin(UserAdmin):
17    list_display = ['username', 'email', 'is_active', 'date_joined']
18    list_filter = ['is_staff', 'is_active']
19
20# Регистрируем модели приложения
21@admin.register(Category)
22class CategoryAdmin(admin.ModelAdmin):
23    list_display = ['name', 'product_count']
24    search_fields = ['name']
25
26@admin.register(Product)
27class ProductAdmin(admin.ModelAdmin):
28    list_display = ['name', 'category', 'price', 'in_stock']
29    list_filter = ['category', 'in_stock']
30    search_fields = ['name', 'description']
31    ordering = ['-created_at']
32
33@admin.register(Order)
34class OrderAdmin(admin.ModelAdmin):
35    list_display = ['id', 'customer', 'total', 'status', 'created_at']
36    list_filter = ['status', 'created_at']
37    readonly_fields = ['created_at', 'updated_at']

Объяснение: Полный пример демонстрирует организацию файла admin.py с удалением стандартных моделей, их кастомизацией и регистрацией моделей приложения с различными настройками.
